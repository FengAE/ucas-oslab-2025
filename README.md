### Task 1&2

需要掌握`e1000`的各个寄存器的使用



**CMD (Command, 8位)**: 控制 DMA 行为的命令字段 。

​	**DEXT =**0，指示使用 Legacy 格式 。

​	**EOP**: 置 **1** 表示这是数据包的最后一个帧

**STA (Status, 4位)**: 硬件回写的状态字段。

​	**DD**: 硬件置 **1** 表示该帧发送完毕+




**Status (8位)**: **由硬件填写**，报告接收状态。

​	**DD**: 硬件置 **1** 表示已经把收到的数据放入缓冲区，并处理完了这个描述符，软件可以来取数据了 。

​	**EOP**: 表示这是数据包的最后一帧 。





​	**TX：CPU 往 NIC 推数据（CPU 是生产者）**
​			**RX：NIC 往 CPU 推数据（NIC 是生产者）**



| 寄存器 | 谁写 | 含义                     |
| ------ | ---- | ------------------------ |
| TDT    | CPU  | 下一个“我新增数据”的位置 |
| TDH    | NIC  | 下一个“我要发送”的位置   |



| 寄存器 | 谁写 | 含义                           |
| ------ | ---- | ------------------------------ |
| RDH    | NIC  | 下一个要写的 descriptor        |
| RDT    | CPU  | 最后一个“可写给 NIC”的 descrip |





### Task 3

**网卡四级中断流程**

**Level 1**

网卡外设发送中断，跳转到中断处理入口，根据`scause`跳转到对应的中断处理函数

**Level 2**

依据中断号为`IRQC_S_EXT`跳转到`handle_irq_ext`

**Level 3**

通过 `plic_claim`来获取是哪一 个外设发出了中断。`plic_claim` 会查询到E1000网卡外设 的ID 号，并跳转到E1000中断处理入口

PLIC：选取优先级最高、中断被使能中断，并设置xIP.xEIP位，进行iterrupt notification（本实验将EIP直接连接到中断目标的exip）

目标接收到时，进行读取claim寄存器。选最高优先级ID，**清除ID**，并发送给中断目标（ID=0：无中断）

匹配ID后，读取ICR寄存器，查询中断信息并处理

完毕后：写入ID到complete寄存器，否则会重复触发中断！

**Level 4**

读取 E1000 的中断原因寄存器ICR，判断是哪一种中断

ICR：记录中断触发的独热码。ICR+IMS某一位=1，则pending bit置1，向处理器发送中断信号

ICS：可以设置ICR中断位

IMS：掩码，某位=1表示启动

任何写0均无效！而写1可以清除其他位



本实验设计的两种中断：

TXQE：TDh==TDT触发，需要分配更多发送描述符（发送队列满）

RXDMT0：当硬件所持有的接收描述符占全部的比例小于一半时（(RDT - RDH) ≤ RXDMT0 阈值），则触发该中断，提醒软件应当及时取走已接收的数据帧，并为硬件分配更多的接收描述符





### Task 4

三种报头：

1. DAT：数据发送方发送一个含有数据的报文，设置seq。
2. ACK：数据接收方确认收到数据，需要设置seq，表示序号小于该seq的数据包已 经收到。 
3. RSD：数据接收方请求重发一个数据包，seq设置为该数据包的序号。发送方必须 响应每一个RSD包

由于实验板卡不支持网卡`E1000_ICR_RXT0`中断，如果沿用此前的网卡中断会出现唤醒丢失，故而在本task不进行block，而是通过sleep函数定时唤醒
